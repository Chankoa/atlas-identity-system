# Renderer - Firefly

## Rôle

Firefly sert à produire une direction artistique, un moodboard ou des intentions visuelles à partir du contenu Markdown.

Il ne sert pas à rédiger la candidature et ne remplace pas les fichiers `jobs/[candidature]/`.

## Entrées Markdown

- `jobs/[candidature]/firefly-brief.md`
- `jobs/[candidature]/analysis.md`
- `design-system/colors.md`
- `design-system/typography.md`
- `design-system/image-direction.md`

## Usage recommandé

1. Copier le prompt principal depuis `firefly-brief.md`.
2. Copier le prompt négatif dans le champ prévu si l'outil le permet.
3. Générer plusieurs pistes visuelles.
4. Sélectionner uniquement les éléments compatibles avec le ton du dossier.
5. Reporter les choix retenus dans `jobs/[candidature]/rendering-workflow.md`.

## Sorties attendues

- Moodboard ou intention visuelle.
- Image d'ambiance si utile.
- Références de matière, rythme, couleurs et composition.

## Limites

- Ne pas utiliser de logo ou d'identité existante sans autorisation.
- Ne pas créer de faux portrait.
- Ne pas intégrer de texte final dans une image.
- Ne pas considérer une image générée comme une preuve ou une réalisation.
